# KeyKiller Launcher 

GUI-лаунчер для `kk.exe` з підтримкою чанків, відстеженням прогресу  
та візуальним вибором діапазону. Написано на Python / tkinter (без додаткових залежностей).

**Вимоги:** Python 3.8+, Windows 10/11  
**Запуск:** `python launcher_kk_ua.py`  
**Тестувалось на:** NVIDIA RTX A4000 (48×128 ядер, SM86), Windows 10 x64, ~1560 Mkey/s

Доступний у трьох мовах:

| Файл | Мова |
|------|------|
| `launcher_kk_ru.py` | 🇷🇺 Російська |
| `launcher_kk_en.py` | 🇬🇧 Англійська |
| `launcher_kk_ua.py` | 🇺🇦 Українська |

## Можливості

### Вибір пазлу
- Випадаючий список з **48 невирішеними пазлами** (#71 – #129)
- При виборі пазлу **автоматично підставляються**: Bitcoin-адреса, межі hex-діапазону,  
  шкала слайдера, поля Start/End, підпис bits
- При зміні пазлу запитується очищення історії пройдених ділянок

### Шкала діапазону
- **Подвійний слайдер** на весь діапазон пазлу (0% – 100%)
- **Кнопка замку 🔒** — з'єднує обидва повзунки, вони рухаються разом  
  зі збереженням ширини чанку (зручно для прокрутки вікна фіксованого розміру)
- **Поля Start % / End %** — введи точні відсотки і натисни Застосувати
- **Поля Start HEX / End HEX** — введи точні hex-значення вручну
- Всі чотири поля (два % + два HEX) та слайдер **повністю синхронізовані** —  
  зміна будь-якого миттєво оновлює всі інші
- Пройдені ділянки відображаються **зеленими смугами** на шкалі
- Кнопка **Очистити пройдені** скидає зелену історію

### Режими пошуку
- **Послідовний** — сканує від Start до End по порядку; підтримує  
  збереження/продовження (флаг `-b`)
- **Випадковий (-R)** — вибирає випадкові позиції в межах чанку;  
  відновлення не потрібне

### Блок основних налаштувань
- Шлях до `kk.exe` з кнопкою вибору файлу
- Поле ID пристрою GPU
- Чекбокс `-b` Resume (для послідовного режиму)
- **Зберегти конфіг** / **Завантажити** — зберігає всі налаштування у `kk_config.json`

### Блок параметрів / Інфо
- Відображає в реальному часі: поточний пазл, адресу, активний чанк, режим, повну команду

### Тестовий блок
- Окрема секція для перевірки пайплайну знаходження ключів на **відомій адресі**
- Ручне введення: Адреса, Start HEX, End HEX
- Вибір режиму: Послідовний або Випадковий
- Кнопки **▶ ПРОТЕСТУВАТИ** / **■ СТОП**
- Окремий конфіг зберігається у `kk_test_config.json` (завантажується автоматично)
- При знаходженні ключа: спливаюче вікно + файл `TEST_FOUND_<мітка_часу>.txt`

### Виявлення знайденого ключа
- Фоновий watcher стежить за `found.txt` за **часом модифікації**  
  (не за розміром — уникає хибних спрацювань коли файл вже існує)
- При виявленні: спливаюче вікно, запис у журнал, копія у `FOUND_<мітка_часу>.txt`
- Пройдена ділянка автоматично позначається на шкалі після успішного запуску

### Поведінка при запуску
- `kk.exe` запускається в **окремому CMD-вікні** — весь вивід GPU видно там
- Журнал лаунчера показує: мітки часу, використану команду, події старту/зупинки, знайдені ключі
- Кнопка **СТОП** примусово завершує процес kk.exe через `taskkill /F /T`

## Файли, що зберігаються

| Файл | Вміст |
|------|-------|
| `kk_config.json`      | Основні налаштування: шлях до exe, пазл, GPU, режим, чанк, % значення |
| `kk_test_config.json` | Налаштування тестового блоку: адреса, діапазон, режим |
| `kk_history.json`     | Пройдені ділянки на шкалі (окремо для кожного пазлу) |
| `kk_progress.json`    | Поточна сесія: пазл, адреса, чанк, режим, час початку |
| `found.txt`           | Записується kk.exe при знаходженні ключа |
| `FOUND_<ts>.txt`      | Копія, зроблена лаунчером в момент виявлення |
| `TEST_FOUND_<ts>.txt` | Копія, зроблена лаунчером після тестового запуску |

## Швидкий старт

1. Поклади `launcher_kk_ua.py` у ту саму папку де знаходиться `kk.exe`
2. Запусти: `python launcher_kk_ua.py`
3. Вибери пазл зі списку (наприклад **Puzzle 71**)
4. Встанови свій чанк за допомогою слайдера або полів HEX/%
5. Вибери режим: Послідовний або Випадковий
6. Натисни **▶ ЗАПУСТИТИ** — kk.exe відкриється в окремому CMD-вікні
7. Після завершення чанк автоматично позначається зеленим на шкалі

### Тест перед реальним пошуком

Скористайся **Тестовим блоком** щоб перевірити що все працює:
- Адреса: `1PWCx5fovoEaoBowAvF5k91m2Xat9bMgwb`
- Start HEX: `400000000`
- End HEX: `7FFFFFFFF`
- Режим: Послідовний

Ключ знайдеться менш ніж за 1 секунду на RTX A4000.

## Примітки

- Підтримуються тільки **Compressed P2PKH** адреси (пазли #51 і вище)
- Пазли #1–#50 використовують Uncompressed ключі — kk.exe їх не знайде
- Конфіг-файли у форматі JSON — можна редагувати вручну за потреби
- Всі три мовні версії використовують спільні конфіг-файли
